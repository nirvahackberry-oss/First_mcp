# Work Report MCP

## Purpose

This MCP server helps automate daily work reporting.

Instead of manually opening Excel and writing daily updates, Cursor can:

1. Read changes from Project A and Project B.
2. Analyze Git changes.
3. Generate a professional work summary.
4. Append rows into the daily report Excel file.

---

# Folder Structure

```text
D:\
│
├── WorkReportMCP\
│   ├── server.py
│   ├── config.py
│   ├── git_tools.py
│   ├── excel_tools.py
│   ├── report_generator.py
│   └── README.md
│
├── ProjectA\
│   └── .git
│
├── ProjectB\
│   └── .git
│
└── Daily_Task_Report.xlsx
```

---

# Excel Format

The Excel file must contain the following columns:

| SR NO. | DATE | PROJECT NAME | TASK /MODULE | DESCRIPTION | SCREENSHORT | DUE DATE | STATUS |
| ------ | ---- | ------------ | ------------ | ----------- | ----------- | -------- | ------ |

# Configuration

Edit `config.py`.

```python
PROJECTS = {
    "project_a": r"D:\ProjectA",
    "project_b": r"D:\ProjectB"
}

EXCEL_FILE = r"D:\Daily_Task_Report.xlsx"
```

Replace the paths with actual locations.

# Cursor MCP Configuration

Add to Cursor MCP settings:

```json
{
  "mcpServers": {
    "work-report": {
      "command": "D:\\WorkReportMCP\\.venv\\Scripts\\python.exe",
      "args": [
        "D:\\WorkReportMCP\\server.py"
      ]
    }
  }
}
```

Restart Cursor after saving.

# Available Tools

## scan_project

Returns today's project changes.

Example:

```text
scan_project("project_a")
```

---

## create_report

Adds a row to Excel.

Example:

```text
create_report(
    project_name="Project A",
    task_module="Authentication",
    description="Implemented login validation and token refresh handling"
)
```

---

# Recommended Workflow

Open Cursor.

Ask:

```text
Generate today's report for project_a and update Excel.
```

Cursor should:

1. Call scan_project().
2. Analyze changes.
3. Create summary.
4. Call create_report().
5. Append row to Excel.

---

# Multiple Projects

If both projects were worked on:

```text
Generate today's report for all projects and update Excel.
```

Expected result:

Project A -> Row 1

Project B -> Row 2

Both rows use the same date.

---

# Notes

* Project folders do NOT need to be opened in Cursor.
* MCP reads repositories directly from configured paths.
* Git diff is preferred over commit history because work may not be committed yet.
* One row is typically generated per project per day.
* If no changes are detected, no row should be created automatically.
